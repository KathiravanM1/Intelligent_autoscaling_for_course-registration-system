Queue-Theoretic Formulas (M/M/N Model)

Service Rate (μ)
μ = 1 / latency
How many requests one replica can process per second. If latency = 0.1s, then μ = 10 req/s per replica.
Utilization (ρ)
ρ = λ / (N × μ)
How busy the system is. λ = arrival rate, N = replicas, μ = service rate. ρ must be < 1.0 for stability. ρ = 0.8 means 80% busy.
Steady-State Probability (P₀)
P₀ = 1 / [ Σ(n=0 to N-1) (λ/μ)ⁿ/n! + (λ/μ)ᴺ / (N! × (1-ρ)) ]
Probability that the system is completely idle (zero requests being served). Used to compute queue length.
Mean Queue Length (Lq)
Lq = P₀ × (λ/μ)ᴺ × ρ / (N! × (1-ρ)²)
Average number of requests waiting in queue (not yet being served). High Lq = system is overloaded.
Mean Response Time (W)
W = (Lq / λ) + (1 / μ)
Average total time a request spends in the system (waiting + service). SLA bounds: Tmin=0.02s, Tmax=0.08s. W > Tmax → SCALE OUT. W < Tmin → SCALE IN.
MSO Scale-Out Target
ratio = (λ₂ - λ₁) / λ₁  |  target = N + ceil(ratio × N)
When traffic increases by X%, add X% more replicas proportionally. Example: 4 replicas, traffic up 50% → add 2 → target = 6.
Scale-In Conditions (all 3 must hold OR delta-lambda decreasing for 3 cycles)
W < Tmin AND rho < 0.3 AND (lambda/N) < 0.3 x cap_per_replica
OR: delta-lambda < 0 (arrival rate decreasing) for 3 consecutive cycles
Scale in when response time is well below SLA, utilization is low, and each replica handles less than 30% capacity — OR when arrival rate is consistently decreasing — for 3 consecutive cycles.
